Referensi pseudocode untuk semua proses posting di sistem warehouse. Klik tab manapun untuk lihat detail — termasuk tabel apa yang di-INSERT, UPDATE, dan bagaimana transaction berjalan.

Cara baca pseudocode ini
BEGIN TRANSACTION — semua operasi di dalam blok ini bersifat atomic (semua berhasil atau semua batal)
FOR UPDATE — row-level lock, mencegah dua user menulis saldo bersamaan
INSERT INTO t_stock_mutation — append only, tidak pernah update/delete
UPDATE t_stock_balance — saldo real-time yang selalu di-update setiap ada pergerakan stok
ROLLBACK — batalkan semua perubahan jika ada error
